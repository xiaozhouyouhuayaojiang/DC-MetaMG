**Internal flow of main.py:**
1. `set_seed(args.seed)`
2. `smiles, splits = get_data(args)` — loads `edges.txt` + `DrugSmile.csv` from `data_path`, builds bipartite graph, runs `RandomLinkSplit` (80/20 train/test, undirected, negative sampling ratio 1.0), and **persists the split to `<data_path>/splits.pt`** (note: the `if False and os.path.exists(...)` check in `util.get_data` means the cached splits.pt is currently never loaded — it always regenerates and overwrites `splits.pt`, even if one already exists).
3. `model = MetaMG(args, smiles, args.embedding_size, 1, splits['train'])` — builds the model, which in `InitFeature()` loads `GIP_Topo_drug.txt`, `Gaussian_sim_<dataset>.txt`, `Feature_<dataset>_SQ.txt` from `data_path` as initial embeddings, and internally computes drug-molecule embeddings via RDKit/DGL from SMILES (`DrugMG`).
4. `trainer = Trainer(args, model, splits)`; `trainer.train(splits['train'])` runs the training loop for `args.epochs` epochs, then calls `_save_embeddings()` and `_save_edge_splits()`.

**Output produced by running `main.py` (Stage 1 deliverables):**
- Directory `GetFeature/source/<dataset>/` is created (relative to CWD, i.e. relative to wherever `main.py` is run from — must run from `GetFeature/` directory for paths to resolve as documented).
- Files written:
  - `Drug_GT.txt` — coarse-grained drug embeddings (space-delimited via `np.savetxt`)
  - `NcRNA_GS.txt` — coarse-grained ncRNA embeddings
  - `Drug_MG.txt` — fine-grained drug embeddings
  - `NcRNA_SQ.txt` — fine-grained ncRNA embeddings
  - `edge_splits.pt` — a `dict` saved via `torch.save`, containing:
    - `train_pos_edge_index`, `train_neg_edge_index` (always present)
    - `test_pos_edge_index`, `test_neg_edge_index` (present if test split exists)
    - Each is a `torch.LongTensor` of shape `[2, E]` (row 0 = drug idx, row 1 = ncRNA idx, offset by `n_Drug`)
- Also as a side effect: `<data_path>/splits.pt` is (re)written by `get_data()` — this is the PyG train/test `Data` object split (different from `edge_splits.pt`; this one is GetFeature's own internal link-split cache, not consumed by DC_Prediction).

### 2.2 `GetFeature/trainer.py` — High-Level Flow

- `Trainer(args, model, splits)` wraps the `MetaMG` model.
- `.train(train_data)`:
  - Creates Adam optimizer (`lr=1e-3`, `weight_decay=5e-4`) over `model.parameters()`.
  - Loops `epochs` times calling `_train_epoch`, which does forward pass via `model.calculate_loss(train_data)`, backprop, gradient clipping (`clip_grad_norm_`), optimizer step. Prints loss every 10 epochs.
  - After training completes: calls `_save_embeddings()` (writes the 4 `.txt` embedding files into `source/<dataset>/`) and `_save_edge_splits()` (writes `edge_splits.pt` into the same folder).
- Trains a **single model instance**, no k-fold/cross-validation in this stage — that happens later in DC_Prediction.

### 2.4 Dynamic Meta-path Selection — Separate Preprocessing Step (Run BEFORE GetFeature/main.py)

This is **not invoked by `GetFeature/main.py`**. It is a standalone preprocessing stage whose outputs (`MetaPath_drug*.txt`, `MetaPath_miRNA.txt` / `MetaPath_lncRNA.txt`, similarity matrices, etc.) feed into `GetFeature/data/<dataset>/DATA_DMP/` and other `data/<dataset>/` files. It must be run first, manually, before `GetFeature/main.py`.

**`Dynamic Meta-path Selection/MiRNA/main.py`** (43 lines, full content read):
- Imports `getValue` (unused directly in visible code) and `set_seed, getMetaPath` from local `util` package.
- `set_seed(48)` — hardcoded seed, independent of GetFeature's seed arg.
- Loads `data/array_DD.txt` (drug-disease matrix) and `data/array_MD.txt` (miRNA-disease matrix) — relative to this folder, i.e. `Dynamic Meta-path Selection/MiRNA/data/`.
- Computes drug meta-path (`DDI`) via `getMetaPath(num=num_D, object_disease_matrix=drug_disease_matrix, correlation_file='data/correlation_Drug.txt', max_hops=3, alpha=2.0, beta=1.0)`, writes to `data/MetaPath_drug.txt` (relative to this folder).
- Computes miRNA meta-path (`MMI`) similarly with `max_hops=2`, `correlation_file='data/correlation_MiRNA.txt'`, writes to `data/MetaPath_miRNA.txt`.
- **Must be run from inside `Dynamic Meta-path Selection/MiRNA/` directory** (uses relative paths `data/...`).
- Output paths are local to this subfolder (`data/MetaPath_drug.txt`, `data/MetaPath_miRNA.txt`) — these would then need to be manually copied/placed into `GetFeature/data/MiRNA/DATA_DMP/` for `main.py`'s `get_edge_index(mode='MetaPath')` to find them (matching the naming-mismatch issue noted in 2.3).

**`Dynamic Meta-path Selection/LncRNA/main.py`** (43 lines, full content read):
- Same structure, `set_seed(48)`.
- Loads `data/array_DD.txt` and `data/array_LD.txt` (lncRNA-disease matrix) relative to this folder.
- Drug meta-path: same params as MiRNA version, but writes directly to `../../data/LncRNA/DATA_DMP/MetaPath_drug.txt` — i.e. an **absolute-relative path that writes straight into `GetFeature/data/LncRNA/DATA_DMP/`** (two levels up from `Dynamic Meta-path Selection/LncRNA/`). This matches the LncRNA `DATA_DMP/MetaPath_drug.txt` file confirmed present in section 2.3.
- lncRNA meta-path: `max_hops=2`, `correlation_file='data/correlation_lncRNA.txt'`, writes to `../../data/LncRNA/DATA_DMP/MetaPath_lncRNA.txt` — also matches confirmed file `MetaPath_lncRNA.txt`.
- **Inconsistency to note in README:** the LncRNA version writes its outputs directly into `GetFeature/data/LncRNA/DATA_DMP/`, while the MiRNA version writes outputs locally into its own `data/` folder (requiring a manual copy step into `GetFeature/data/MiRNA/DATA_DMP/`). The README should explicitly call out this asymmetry as a manual step for the MiRNA pipeline.
- Also contains larger `GetMatrix_Drug/` and `GetMatrix_LncRNA/` (or `GetMatrix_MiRNA/`) sibling subfolders with their own scripts (`jaccard_similarity.py`, `gaussian_similarity.py`, `Get_PathMatrix/PRA/PRA.py`, `Test/` scripts for `ClearCommon.py`/`CreateIndex.py`/`GetArr.py`) that appear to be earlier preprocessing steps generating the similarity/array files (`array_DD.txt`, `DD_ARR.txt`, `Jaccard_sim_Drug.txt`, `Gaussian_sim_Drug.txt`, `Topo_drug.txt`, `GIP_Topo_drug.txt`, `drug_embeddings.csv`, etc.) consumed by `Dynamic Meta-path Selection/<dataset>/main.py` and ultimately by `GetFeature/data/<dataset>/`. These were not read in detail (out of scope per the request, which only asked for `main.py` files), but should be mentioned as an even-earlier optional preprocessing layer if the user wants to regenerate similarity matrices from scratch rather than using the provided `.txt`/`.csv` files already present in the repo.

### 3.2 `DC_Prediction/util.py` — load_embeddings() and load_edge_splits()

```python
def load_embeddings(dataset: str = 'MiRNA'):
    base = os.path.join('data', dataset)
    embedding1 = np.loadtxt(os.path.join(base, 'Drug_GT.txt'), dtype=float, delimiter=" ")
    embedding2 = np.loadtxt(os.path.join(base, 'NcRNA_GS.txt'), dtype=float, delimiter=" ")
    embedding3 = np.loadtxt(os.path.join(base, 'Drug_MG.txt'), dtype=float, delimiter=" ")
    embedding4 = np.loadtxt(os.path.join(base, 'NcRNA_SQ.txt'), dtype=float, delimiter=" ")
```
